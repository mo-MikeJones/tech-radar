## Summary
Emerging practices for effective human-AI pair programming, including how to structure the collaboration, divide cognitive tasks and maintain code quality when working alongside AI.

## Why we're interested
The shift to AI pair programming requires new habits and practices beyond simple tool adoption. Early definition of these practices shapes how engineers work going forward.

## Current status
Practices being documented through observation of effective AI users. Early guidance published. Still evolving based on ongoing feedback.

## Evidence & feedback
Engineers who adopt deliberate AI pair programming practices report more productive AI collaboration compared to ad-hoc usage. Structured prompting and incremental review are the most effective patterns identified.

## Known risks / concerns
Without clear practices, AI pair programming can lead to passive acceptance of suggestions and reduced code ownership.

## Recommended audience
All engineers using AI coding tools. Particularly valuable for senior engineers mentoring others in AI-augmented development.

## Next steps
Publish initial practices guide. Run team workshops on effective AI pair programming. Iterate based on community feedback.
