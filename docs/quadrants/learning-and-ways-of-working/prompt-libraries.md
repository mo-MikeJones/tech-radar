## Summary
Curated collections of reusable, high-quality prompts for common development tasks, shared across engineering teams to improve consistency and reduce experimentation overhead.

## Why we're interested
Writing effective prompts is a skill that takes time to develop. Shared libraries accelerate teams and encode proven approaches in a reusable form.

## Current status
Initial library of ~50 prompts available. Growing through community contributions. Actively used by several teams.

## Evidence & feedback
Teams using shared prompts report more consistent AI outputs. Most popular prompts cover code review, test generation and documentation templates.

## Known risks / concerns
Prompts need validation as AI models are updated. Low-quality contributions can undermine trust in the library if curation is insufficient.

## Recommended audience
All engineers using AI coding tools. Prompt authors and contributors looking to share effective patterns.

## Next steps
Implement a curation and rating system. Integrate with the IDE via a prompt picker extension. Track usage to identify the most valuable prompts.

Developer Experience Team
