## Summary
A catalogue of proven architectural patterns for designing effective AI agent systems, including patterns for task decomposition, tool use, memory and multi-agent coordination.

## Why we're interested
Agent development is a new discipline where common pitfalls can be avoided through established patterns. A shared pattern catalogue raises the quality of agent implementations across the organisation.

## Current status
Initial catalogue of 10 patterns documented. Being expanded through community contribution and frontier team findings.

## Evidence & feedback
Teams using documented patterns report fewer architectural mistakes in agent design. Most valuable for teams new to building AI agent systems.

## Known risks / concerns
The field is evolving rapidly. Patterns may need revision as best practices mature. Avoid treating patterns as rigid prescriptions.

## Recommended audience
Engineers designing and building AI agent systems. Architects assessing agentic architectures for production readiness.

## Next steps
Publish the pattern catalogue on the developer portal. Run workshops to introduce patterns. Plan a quarterly review cycle.
