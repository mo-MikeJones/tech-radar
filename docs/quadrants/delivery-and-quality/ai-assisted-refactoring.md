## Summary
Using AI tools to support large-scale code refactoring, including renaming, extracting components, migrating patterns and improving readability.

## Why we're interested
Refactoring is high-value but time-consuming work. AI can accelerate repetitive structural changes while maintaining correctness.

## Current status
Used in targeted refactoring projects. Particularly effective for well-defined, repeatable transformations.

## Evidence & feedback
Teams report significant time savings on pattern-based refactoring. More complex semantic refactoring still requires heavy human guidance.

## Known risks / concerns
AI refactoring tools may miss subtle context. Each transformation must be reviewed and tested before committing to ensure behaviour is preserved.

## Recommended audience
Engineers conducting planned refactoring sprints. Architects driving large-scale codebase modernisation.

## Next steps
Document patterns where AI refactoring is most effective. Create a library of reusable prompts for common refactoring scenarios.
